types of resposes:
200 OK
✔ Request successful
✔ Data fetched / login success / normal response
res.status(200).json({ message: "Login successful", user });

Use when:
Login successful
Data fetched
Normal API success

201 Created
✔ New resource created (signup, create post)
res.status(201).json({ message: "User registered" });

Use when:
User signup
Record saved in DB
204 No Content
✔ Success but no response body

res.status(204).end();
Use when:
Logout
Delete request


❌ 4xx — CLIENT ERROR (frontend ki galti)

400 Bad Request
❌ Missing / invalid input
res.status(400).json({ message: "Email and password required" });
Use when:
Form incomplete
Invalid data format

401 Unauthorized
❌ User logged in nahi hai
res.status(401).json({ message: "Please login first" });
Use when:
JWT missing
Session expired
User not authenticated

403 Forbidden
❌ Login hai but permission nahi
res.status(403).json({ message: "Access denied" });
Use when:
User trying admin route
Role based access denied
404 Not Found
❌ Resource nahi mila
res.status(404).json({ message: "User not found" });
Use when:
Wrong API
User does not exist
409 Conflict
❌ Already exists
res.status(409).json({ message: "Email already registered" });

Use when:
Duplicate email
Username already taken
422 Unprocessable Entity

❌ Validation error
res.status(422).json({ message: "Password too short" });

Use when:
Data format correct but rules break
💥 5xx — SERVER ERROR (backend ki galti)
500 Internal Server Error
❌ Backend crash / DB error
res.status(500).json({ message: "Something went wrong" });

Use when:
Try/catch error
Database down
503 Service Unavailable
❌ Server down / maintenance
res.status(503).json({ message: "Server busy" });
